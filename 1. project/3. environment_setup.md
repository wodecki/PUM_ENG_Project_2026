# Environment Setup

## Option 1: Google Colab (Recommended)

Google Colab is the primary supported environment. No local installation required.

### Steps

1. **Upload data files** to Google Drive:
   ```
   My Drive/
   └── PUM/
       └── 2. data/
           ├── customers.csv
           └── transactions.csv
   ```

2. **Open the starter notebook** in Colab:
   - Upload `3. notebooks/mpN_starter.ipynb` to Colab, OR
   - Upload to Google Drive and open from there

3. **Run the first cell** (reproducibility locks):
   ```python
   import sklearn, pandas as pd, numpy as np
   # Version assertions will pass on standard Colab
   ```
   If the assertions fail, see Troubleshooting below.

4. **Run the second cell** (data loading):
   - The notebook auto-detects Colab and adjusts the data path
   - MD5 fingerprint check verifies your data files are correct

### Colab Data Path

The notebooks automatically detect Colab and set:
```python
DATA_DIR = Path("/content/drive/MyDrive/PUM/2. data")
```

Make sure your files are at this exact path, or modify the path in the notebook.

### Mounting Google Drive

If prompted, mount your Drive:
```python
from google.colab import drive
drive.mount('/content/drive')
```

## Option 2: Local Setup

### Prerequisites

- Python 3.10 or later
- `uv` package manager ([install guide](https://docs.astral.sh/uv/getting-started/installation/))

### Steps

1. **Clone or download** the repository

2. **Create virtual environment and install dependencies**:
   ```bash
   cd PUM_ENG
   uv venv
   uv pip install pandas numpy scikit-learn matplotlib seaborn jupyter
   ```

3. **Verify library versions**:
   ```bash
   uv run python -c "
   import sklearn, pandas, numpy
   print(f'sklearn:  {sklearn.__version__}')
   print(f'pandas:   {pandas.__version__}')
   print(f'numpy:    {numpy.__version__}')
   "
   ```
   Expected: sklearn 1.4.x–1.5.x, pandas 2.x, numpy <2.0

4. **Start Jupyter**:
   ```bash
   uv run jupyter notebook "3. notebooks/"
   ```

5. **Open the starter notebook** and run the first two cells to verify setup

### Local Data Path

When running locally, the notebooks set:
```python
DATA_DIR = Path("../2. data")
```

This works if you launch Jupyter from the `3. notebooks/` directory.

## Troubleshooting

### "AssertionError: scikit-learn version..."

Your scikit-learn version is outside the supported range.

**Colab fix:**
```python
!pip install scikit-learn==1.5.2
# Then restart the runtime: Runtime → Restart runtime
```

**Local fix:**
```bash
uv pip install scikit-learn==1.5.2
```

### "AssertionError: numpy version..."

You have numpy 2.x installed. Some sklearn functions behave differently with numpy 2.

**Fix:**
```python
!pip install numpy==1.26.4
# Restart runtime
```

### "AssertionError: customers.csv MD5 mismatch"

Your data file is corrupted or from a different version.

**Fix:** Re-download `customers.csv` from the course materials. The correct MD5 hash is `c016cd4bfc36ac8d0a99bb6a3d55fa44`.

### "FileNotFoundError: data directory not found"

The notebook can't find your data files.

**Colab:** Make sure files are at `My Drive/PUM/2. data/customers.csv`

**Local:** Make sure you're running Jupyter from the `3. notebooks/` directory, so `../2. data/` resolves correctly.

### "ModuleNotFoundError: No module named 'seaborn'"

**Colab:**
```python
!pip install seaborn
```

**Local:**
```bash
uv pip install seaborn
```

### My numbers don't match the expected answers

1. Check that you're using `random_state=42` everywhere
2. Check that you followed the exact 12-step preprocessing order in MP2
3. Run `3. notebooks/scripts/verify_checkpoint.py` to compare your MP2 output against the golden baseline
4. If your output is close but not exact, check:
   - Did you sort columns alphabetically after encoding?
   - Did you use `drop_first=False` in `pd.get_dummies()`?
   - Did you convert boolean dummy columns to int?
   - Did you remove outliers from BOTH X and y (and gender_series)?

### Google Colab session disconnects

Colab sessions time out after ~90 minutes of inactivity. Save your work frequently. Consider downloading your notebook periodically.
